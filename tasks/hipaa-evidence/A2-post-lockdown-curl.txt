Interpretation
--------------
All four objects return HTTP 401 + Postgres error code 42501 ("permission denied
for view/table"). This is the strongest possible failure mode: the request never
reaches RLS because the `anon` role no longer has SELECT privilege on the objects.

Pre-lockdown, these endpoints would have returned HTTP 200 with full row data
(see audit doc: 7 rows on flagged_messages_view, 469 rows on admin_ai_audit_trail,
90 rows on consultation_bookings).
